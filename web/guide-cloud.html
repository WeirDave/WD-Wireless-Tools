WD Cloud Manager

Manage Ekahau cloud sites and local project folders side by side. Sync, merge, and organize your wireless survey projects.

Getting Started

Step 1
Log In
Click Log In to authenticate with Ekahau Cloud. Your browser will open to complete the login flow.
Step 2
Set Local Folder
Open Settings and set your local folder path — where your .esx project files are stored on disk.
Step 3
Browse
The Sites tab shows the ledger view — cloud sites on the left, local folders on the right.
Step 4
Manage
Rename, delete, create, or merge projects directly from the ledger. Changes sync immediately.
Tip: After logging in, Cloud Manager remembers your session. You only need to re-authenticate if your token expires or you log out.

The Ledger View

The main interface is a two-column ledger that shows your Cloud Sites on the left and Local Folders on the right. This makes it easy to see what exists in each location at a glance and spot projects that are only on one side.

  • Cloud Sites (left) — projects stored in Ekahau Cloud, pulled from your account.
  • Local Folders (right) — .esx files found in your configured local folder path.
  • Sites that exist in both locations are shown on the same row when names match.
  • Sites that only exist on one side appear with an empty cell on the other side.
Tip: The ledger auto-refreshes on a configurable interval (see Settings). You can also manually refresh by clicking the refresh button in the toolbar.

Tabs

TabPurpose
SitesThe main ledger view — cloud vs. local side by side. This is where you manage, rename, delete, create, and merge projects.
Project FilesBrowse the individual files within a selected project. Useful for inspecting floor plans, images, and survey data inside an .esx archive.

Managing Sites

Right-click a site or use the action buttons on any row in the ledger to manage it.

Create

Create a new empty site on the cloud side, or a new project folder on the local side. The new entry appears in the ledger immediately.

Rename

Rename a cloud site or local folder directly from the ledger. The change takes effect on the cloud or filesystem right away.

Delete

Remove a cloud site or local folder. You will be prompted to confirm before anything is deleted.

Note: Deleting a cloud site removes it from Ekahau Cloud permanently. Deleting a local folder moves it to the system recycle bin when possible.

Merging Projects

The merge feature lets you compare a cloud site against a local project and sync differences between them.

  1. Select a row where both a cloud site and a local folder exist.
  2. Click Merge to open the merge preview.
  3. The preview shows what has changed — added, modified, or removed files on each side.
  4. Choose which direction to sync: cloud to local, local to cloud, or resolve conflicts individually.
  5. Confirm the merge to apply the changes.

Conflict Resolution

When a file has been modified on both the cloud and local side since the last sync, a conflict is flagged. You choose which version to keep — cloud, local, or skip the file entirely.

Note: Always review the merge preview before confirming. Merging overwrites files on the target side with the selected version.

Settings

Open the Settings panel from the toolbar to configure Cloud Manager.

SettingDescription
Local Folder PathThe directory on your machine where local .esx project files are stored. Cloud Manager scans this folder to populate the right side of the ledger.
Auto-Refresh IntervalHow often the ledger re-fetches cloud sites and re-scans local folders (in seconds). Set to 0 to disable auto-refresh.
Tip: Point the local folder path to the same directory Ekahau AI Pro uses for saving projects. That way the ledger always reflects what Ekahau sees.

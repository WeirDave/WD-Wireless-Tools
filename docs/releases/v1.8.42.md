Bulk operations in Cloud Manager now show a live progress dialog instead of dead-silent minutes-long waits. Every row in your selection gets a status row that flips from ⏸ pending → ⏳ running → ✅ done or ❌ failed as the loop works through them, with a counter, per-item error messages, and a "Stop after current" button.

Suite `v1.8.41 → v1.8.42`. WD Cloud Manager `v3.45 → v3.46`.

## What changed

### Progress dialog for bulk Sync operations

Before: click Sync → on 14 cloud-only projects, wait 90+ seconds with no feedback, get a single toast at the end. If one failed you had no idea which one, and if you were watching a 62MB `.esx` come down you had zero indication it was still moving vs. hung.

Now: a modal appears the moment the loop starts (for 2+ items), listing every row you selected:

```
Uploading 5 .esx files
3 of 5 · 1 failed

✅ CHTN1 - PD - Ste 1.esx
✅ CSMS1-Building E.esx
⏳ CSMS5 - 600 Anton Blvd.esx        ← in progress
⏸  HNVI1 - PD - Phase 00.esx
❌ IRVN2 - 18582 Teller Ave.esx        File not found: bad path

[ Stop after current ]
```

Icons: ⏸ pending, ⏳ running, ✅ done, ❌ failed. Failed rows keep their inline error message so you can tell what actually went wrong without scrolling through toasts. Summary at the top updates after every step.

**Stop after current** button ends the loop cleanly after the currently-running file finishes — no interruption of an in-flight upload/download, but the queue stops advancing. Useful when you realize you picked the wrong site to download into.

Modal shows for **all three** bulk paths that share Sync:

- Rename loops on matched pairs (Sync → / Sync ←)
- Upload loop on local-only `.esx` files (Sync ←)
- Download loop on cloud-only projects (Sync →)

Mixed selections show everything in one flat queue in the order it'll run, so you can watch a "rename 3, upload 2, download 4" mix step through without wondering where you are.

Single-item operations still just show a toast — no modal noise when you're only doing one thing.

## Files changed

- `web/cloud.html` — new `#progressModal` markup (title, summary, per-row `peek-list`, Stop / Close buttons)
- `web/assets/js/cloud.js`:
  - `openBulkProgress(title, items)`, `updateBulkProgress(i, state, detail, done, total, failed)`, `finishBulkProgress()`, `cancelBulkProgress()` — the modal state machine
  - `bulkSync()` refactored to build a flat `plan` array across all three sub-loops and drive the modal from that; each step reports its own state, error message, and running counter
  - `_bulkAbort` flag checked at each iteration so "Stop after current" ends the loop without cancelling the in-flight request
- `web/assets/wd-tools.css` — three new lines (`.progress-icon`, `.progress-name`, `.progress-detail`) for the row layout

- `web/cloud.html` — Cloud Manager `v3.45 → v3.46` (two places)
- `web/pages/hosted-index.html` — suite `v1.8.41 → v1.8.42`

# Fix: Duplicates tab "unknown api" error

Follow-up patch to v1.3.0 / v1.4.1. Clicking the Duplicates tab in Cloud Manager showed a toast reading `Load failed: unknown api: get_duplicates` and the tab body stuck on "Loading…". The backend endpoint was wired up correctly in v1.3.0; the miss was purely on the JavaScript side.

## What changed

`web/assets/js/cloud.js` — added the missing entry to `API_MAP`:

```js
const API_MAP = {
  get_status: ['status', []],
  ...
  reveal_in_explorer: ['reveal_in_explorer', ['path']],
  get_duplicates: ['get_duplicates', []],   // ← was missing
};
```

`pyApi(method, ...args)` looks up the method in this map and throws `unknown api: <name>` when it's absent. The Duplicates tab's `refreshData()` calls `pyApi('get_duplicates')`, so the throw happened as soon as you opened the tab — before any network request was ever made. Server side (`server.py` `CLOUD_ACTIONS` and `tools/cloud_manager.py` `CloudManager.get_duplicates`) was already correct.

## User-visible behavior change

- Duplicates tab now loads for real. Cluster cards populate with the SYDN2 case and any other near-duplicates in your inventory.
- The ≈ badge on rows in Sites and Project Files tabs also starts working (it depends on the same `get_duplicates` call fired in parallel by `refreshDupIndex()`).

## Verification

1. Restart the Flask server if you had it running from a version before v1.3.0 (`Ctrl+C` in the terminal, then double-click `run.bat`) — otherwise the backend endpoint might not be there. If you're on v1.4.0 or v1.4.1 already, no restart needed.
2. Refresh the browser (hard reload with `Ctrl+Shift+R` to bust the cached `cloud.js`)
3. Click the Duplicates tab — cluster cards render, dashboard shows real Clusters / Mixed / Local Only / Cloud Only counts
4. Switch back to Sites — any row that's part of a cluster now shows an amber ≈ badge next to its name

Three fixes from Dave's Duplicates review — plus a release-notes housekeeping tweak.

Suite `v1.8.14 → v1.8.15`. WD Cloud Manager `v3.24 → v3.25`.

## 1. Row color by side, amber outline on extras

The v1.8.14 approach was: matched rows plain, extra rows amber-tinted. Dave: *"we use colors for the rows like we're doing for the local and for the cloud, and then the ones that are different maybe they have a border around them that is yellow to designate the ones that are standing out."*

That separates two orthogonal concepts cleanly — **where** the item lives (cloud/local) and **whether** it needs attention (matched/extra):

- **`.dup-item.side-cloud`** — subtle blue tint (`rgba(30,119,172,0.06)` dark, `0.09` light). Same blue as the "Cloud Sites/Projects" column header.
- **`.dup-item.side-local`** — subtle green tint (`rgba(95,171,79,0.06)` dark, `0.09` light). Same green as "Local .esx".
- **`.dup-item.is-extra`** — bold amber outline (`box-shadow: inset 0 0 0 2px var(--amber)`) on top of the side tint. Uses inset box-shadow instead of `border` so no layout shift and no compensating padding.
- Hover states retain the side identity — cloud rows brighten in blue, local rows brighten in green.

Cluster header keeps its neutral navy/head-bg treatment from v1.8.14 — no more color collision with extras.

## 2. Delete-all-extras button moved out of the dashboard

v1.8.14 put the button in the dashboard filter row. Dave: *"you're interrupting the flow of the filters up there — that looks terrible."*

Removed from the dashboard entirely. The button now lives in its own **action strip** rendered inside the Duplicates view, positioned between the toolbar and the clusters. Amber-bordered, subtle raised background, with a running count summary:

> **6** extras across **6** clusters — matched pairs will not be touched. &nbsp; &nbsp; [🗑 Delete all 6 extras]

Only renders when there are extras to clean up. Only exists on the Duplicates tab.

## 3. Cloud Manager title actually centered

`.topbar` was using `justify-content: space-between` — left/right sections push to the edges, center element sits in the leftover space. When left and right widths differ (hamburger vs email+folder+theme buttons), the center visually drifts.

Fix: `.topbar-left` and `.topbar-right` are now `flex: 1` with `flex-start` / `flex-end` alignment. The center is `flex: 0 0 auto` and sits in a genuinely centered slot regardless of flank width.

## 4. Release notes housekeeping (this file onward)

Dave noticed the GitHub release page was showing a duplicate title — `--title` sets the release header, and the notes body was also opening with a `# v1.8.X — …` heading. Two titles side by side.

From v1.8.15 forward, release notes markdown starts directly with the description body; `--title` is the sole source of the title. Prior release notes files are left alone.

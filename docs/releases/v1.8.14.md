# v1.8.14 — Duplicates polish: readable metadata, global cleanup, distinct headers

Four fixes from Dave's usage of the trimmed-down Duplicates view:

1. Size/date faded too aggressively on matched items — hard to compare against the extra.
2. No one-click way to delete every extra across every cluster.
3. Owner wasn't shown on dup items — a cloud extra with no site is hard to judge without knowing who created it.
4. Cluster header didn't visually separate from item rows.

Suite `v1.8.13 → v1.8.14`. WD Cloud Manager `v3.23 → v3.24`.

## 1. Size/date stay readable on matched items

In v1.8.11 I faded the entire matched row to `opacity: 0.55` (name/loc) and `0.65` (size/date). The intent was "these are context" — but it made the size/date columns hard to read against the extra's full-opacity size/date, which is exactly what you need to compare.

Fix: opacity now applies only to name + location. Size, date, and metadata pills stay at full opacity across matched and extra items alike, so scanning down the size column reads cleanly.

## 2. Global "Delete all N extras" button

New button in the explanation banner:

```
🗑 Delete all 6 extras (keep every pair)
```

Amber styling to match the extra-item tint. `dupDeleteAllExtras()` walks every cluster in the current `dupData`, collects everything with `matched === false`, and calls `_bulkDeleteItems` — one confirm, one API sweep. Matched items are never touched.

Only renders when there are extras to delete; the button hides when the count is 0.

## 3. Owner shown on every dup item

Backend `build_duplicates_data` now includes `owner` on both cloud and local items:
- Cloud: pulled from `pr["history"]["createdBy"]` (same source as elsewhere).
- Local: pulled from the cached `_esx_author` (same cache used by the Sites/Projects view; no extra I/O).

Frontend appends the owner after the location on the sub-line: `SNAN2 - 3030 S Susan St, Santa Ana, CA 92704 · dpaine@anduril.com`.

**Concrete impact on the case Dave flagged**: the SNAN2 3030 Cleanroom cluster now shows the extra cloud item's owner explicitly — Dave's own email, size `0 B`, `(no site)`. Enough context to see it's an empty orphan he created himself, safe to delete.

## 4. Distinct cluster header

`.dup-head` was inheriting the cluster's white surface — same background as the item rows, no visual separation. Now:

- Dark mode: `--surface-raised` background, amber `border-left: 4px solid`.
- Light mode: `rgba(217,119,6,0.06)` — a soft amber wash, still with the amber left border.
- Hover: slight background shift so it's obviously clickable.

Reads as a section header, not a row.

## Verified against Dave's account (via API)

- 6 clusters returned, every one Mixed with matched pair + 1 extra.
- Owner populated on every item — including the display-name form (`r david paine iii`) for older local .esx files.
- SNAN2 3030 S Susan Cleanroom extra: `owner=dpaine@anduril.com`, `size=0 B`, `location=(no site)` — dramatically more context than before.

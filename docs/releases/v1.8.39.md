Cloud Manager bulk-toolbar polish. Buttons stay visible but now go visually **disabled** when they can't act on the current selection — no more misleading "select some matched rows first" toast on a screen where rows are clearly selected.

Suite `v1.8.38 → v1.8.39`. WD Cloud Manager `v3.42 → v3.43`.

## The problem

v1.8.38 made every bulk-toolbar button always-visible on its tab, on the theory that a stable toolbar beats one that flickers as you check boxes. That's still the right call — but the trade-off was that clicking a button that couldn't act on the current selection just fired an info toast. For **Sync →** / **Sync ←** in particular, the toast said *"select some matched rows first"*, which is deeply confusing when you have 5 local-only rows checked and are staring at your selection on screen.

## The fix

`updateBulkBar()` now counts *what kind* of things are selected and sets `button.disabled = true` on any button whose action doesn't apply. Native browser disabled styling ([`.btn:disabled` in wd-tools.css:429](web/assets/wd-tools.css:429), `opacity: 0.5; cursor: not-allowed`) handles the visuals, and native `<button disabled>` swallows clicks so the misleading toast never fires.

Signals per button (on the Project Files tab):

| Button | Enabled when… |
| --- | --- |
| **Sync →** / **Sync ←** | selection contains at least one matched pair |
| **Move to site…** | selection contains cloud projects or local `.esx` files |
| **Delete selected** | selection contains at least one cloud-only or local-only row |
| **Compare** (Sites tab only) | selection contains 2+ local folders |

Now if you select five local-only `.esx` files, Sync is greyed out at a glance, and Move + Delete are the two obviously-actionable options. No mystery.

## Files changed

- `web/assets/js/cloud.js` — `updateBulkBar()` rewritten to count pair / cloud-only / local-only / local-folder occurrences in the selection and toggle each button's `disabled` state accordingly. Buttons still hide by *tab* (Compare = Sites, Move = Projects); enable/disable is by *selection*.

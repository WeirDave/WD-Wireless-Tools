## 2. Cloud Manager · Settings modal cleanup

The Settings dialog looked rushed — radio labels were rendering ALL CAPS because a legacy `.modal label` rule intended for form-field labels was hitting the radio wrappers too. Rebuilt as two grouped cards with sentence-case labels, section headers, and a pill-shaped `Every 30 seconds` select that matches the button vocabulary.

Before:

## 3. User guides now follow the app theme

The three guide pages (`/guide`, `/guide-cloud`, `/guide-organizer`) were hardcoded light-mode. They now read `localStorage.wd-theme` before first paint and apply `data-theme="dark"` if the app is dark. Every heading, `<kbd>` pill, tip-box, warning-box, and table has proper dark-mode overrides — no more invisible section headers or blank white pill outlines. ## 4. Quick Walls · Add / Clone / Edit Wall Type — full refresh

This modal was the roughest UX in the suite. Rebuilt end-to-end.

### Unit auto-detect + toggle

Thickness now honors your locale. On US locales it opens in **inches**; everywhere else it opens in **meters**. A small `[in|m]` pill next to the label lets you flip live — the current value converts on the fly. Preference persists across sessions in `localStorage.wd-walls-units`. Internally the `.esx` schema always stores meters, so cross-platform files stay compatible.

### Inline descriptions on every field

No more staring at a blank input wondering what belongs in it. Short subtitle under every label:

- **Name** — Descriptive label shown in Ekahau and on your floor plan.
- **Color** — Distinct color for the plan view — pick something that stands out.
- **Thickness** — Real-world wall thickness. Ekahau uses this with attenuation to model signal loss. **Convert units →** (opens Scale in a new tab)
- **Keyboard Shortcut** — Bind to `[1]-[9]` so you can quickly pick this wall type while drawing.
- **Propagation Properties** — How radio waves interact with this wall at each Wi-Fi band. Higher **attenuation** = more signal loss through the wall. **See guide →**

### Wider, shorter layout

Modal grew from 440 px narrow-column-that-scrolls to **900 px** (capped at `min(90vw, 1366px)`) with a single 4-column top row: Name (wide) / Color / Thickness / Keyboard. Descriptions sit under each cell instead of piling up vertically. Propagation table stays full-width below. Fits comfortably in one viewport without scrolling for the common case.

### Clone button

Every wall-card has a new **⧉** action that opens the Add modal pre-loaded from the source wall type. Name gets `(Copy)` appended (with the appended text preselected so you can type over it), keybind cleared so it can't collide with the source. Every other field — color, thickness (auto-unit-converted), propagation values across all three bands — is copied straight through. Click Add to save the copy.

Rationale: the "Recommended by WD" presets are already well-calibrated. Cloning-then-tweaking is a much faster path to a new wall type than building from scratch.

The clone icon is inline SVG (two overlapping squares, back-at-top-left / front-at-bottom-right — the universal duplicate glyph) so it renders crisply regardless of font, unlike the initial `⧉` Unicode character which was barely visible in Segoe UI.

### Add-wall guide section

`/guide` gained a new **"Adding & Editing Wall Types"** section explaining every field including the unit toggle, band-level propagation properties, and when to use the WD-recommended presets as a starting point.

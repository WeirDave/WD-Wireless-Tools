# Fix: broken assets and navigation on the hosted site

Follow-up patch to v1.4.0. The hosted Quick Walls at [weirdave.github.io/WD-Wireless-Tools/walls](https://weirdave.github.io/WD-Wireless-Tools/walls/) had two visible regressions that didn't affect the desktop app: the logo above the dropzone was missing, and every link in the hamburger menu (Home, Cloud Manager, Squirrel, User Guide) took visitors to dead pages outside the project subpath.

Both come from the same root cause — absolute paths (`/foo`) on a GitHub Pages project site (`weirdave.github.io/WD-Wireless-Tools/`) resolve to the domain root, not the project prefix.

## What changed

### 1. Dropzone logo — CSS `url()`

`web/assets/wd-tools.css` — the `.dropzone-logo` background image used an absolute path:

```css
background-image: url(/assets/quick-walls-v1-560x852.png);  /* before */
background-image: url(quick-walls-v1-560x852.png);          /* after */
```

CSS `url()` values resolve relative to the **stylesheet's own location**, not the page URL. The stylesheet lives next to the image at `/assets/wd-tools.css` on both desktop and hosted, so a bare filename correctly finds the sibling PNG under both. No sed rewrite needed in the build step for this one — the source fix works everywhere.

### 2. Hamburger menu navigation — HTML `href`

`walls.html` and the three `guide*.html` help pages carry absolute nav links like `href="/walls"`, `href="/cloud"`, `href="/"`, etc. Those work on the desktop Flask server (which serves the domain root) but break on GitHub Pages under `/WD-Wireless-Tools/`.

The desktop HTML source is unchanged. The **Pages build step** in `.github/workflows/pages.yml` now sed-rewrites absolute nav hrefs during the copy into `docs-build/`:

- Inside `walls/index.html` (deployed at `/WD-Wireless-Tools/walls/`): `href="/"` → `href="../"`, `href="/walls"` → `href="./"`, `href="/cloud"` → `href="../cloud/"`, `href="/organizer"` → `href="../organizer/"`, `href="/guide"` → `href="../guide.html"`
- Inside `guide*.html` (deployed at `/WD-Wireless-Tools/guide.html` etc.): analogous rewrites with paths appropriate to the root level

Sed order matters — the longer, more specific hrefs like `/walls` and `/cloud` rewrite **before** the bare `/`, otherwise the wildcard would clobber them.

## User-visible behavior change

- The Quick Walls logo now renders above the dropzone on the hosted site, matching desktop.
- Every hamburger-menu link on the hosted site now navigates within the project subpath — Home returns to the landing, Cloud Manager and Squirrel go to their "download the desktop suite" stub pages, and the User Guide opens the built-in help.

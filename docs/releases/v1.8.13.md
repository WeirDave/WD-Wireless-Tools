# v1.8.13 — Live is default-on, Refresh button retired

Dave's reasoning:

> "The refresh button can go away because the live toggle would actually be the default. Multiple people have access to the same cloud, so you don't know if another person is working on a site — always keep it on live because of that."

Live already polls every 30 s (adjustable in Settings) and does exactly what Refresh does. With Live default-on, the standalone Refresh button is redundant clutter.

Suite `v1.8.12 → v1.8.13`. WD Cloud Manager `v3.22 → v3.23`.

## Two changes

1. **Removed the "Refresh" button.** One less thing on the toolbar. If you want an immediate refresh, click the Live pill twice (off then on — starts a fresh countdown).
2. **Live defaults to ON.** `liveWanted()` now returns `true` unless localStorage has `wd-live === 'off'`. A user who explicitly turned it off in a past session still gets the off state — new users and users who've never touched the toggle get Live on.

## Why this is safer than it sounds

- Multi-user cloud: someone else may edit a site while you're looking at it. Stale data is worse than a periodic refetch.
- Live already pauses during risky moments — while a modal is open, while you have rows selected, or while you're actively typing in the search box. So you won't have a refresh overwrite an in-progress selection.
- Explicit-off preference is preserved across sessions. The change is only for first-time / never-toggled users.

## Verified in browser

- Toolbar now: `×  + New Site  Live ● 18s  ⚙`. No Refresh button.
- Fresh session (`localStorage.wd-live === null`) → Live starts on automatically, `.live-on` class applied, countdown visible.

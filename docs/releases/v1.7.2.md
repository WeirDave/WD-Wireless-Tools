# WD Report v1.2 ‚Äî Cover page with insertable logo, tighter print layout

Two things from real use of v1.7.1:

- v1.7.1 had **large whitespace gaps** in the printed PDF ‚Äî the per-section padding stacked and the `@page` margin was generous.
- No cover page ‚Äî every report started right on the AP data. You asked for a **cover with an insertable logo**.

Both fixed.

## Cover page

Toggle it in the sidebar under **Cover page ‚Üí Include cover page** (on by default). When enabled, the report opens with a full-page cover:

- Logo (if uploaded ‚Äî see below)
- `Report ¬∑ AP Placement` brand line (teal, small caps)
- Site name as the large title
- Two big stat blocks: total access points and floor plan count
- Generated date

The cover forces a `page-break-after`, so plans start clean on page 2 of the PDF.

## Insertable logo

New sidebar section with an **Upload logo‚Ä¶** button. Pick any PNG/JPG/SVG/WebP/GIF; the logo is read as a data URL and stored in `localStorage` under `wd-report-logo`. It shows in a small sidebar preview immediately and survives reloads ‚Äî no need to re-pick every time.

Only if the browser refuses to save (private-mode / full storage) do you get a warn toast saying the logo is session-only for this run.

A **Remove** button clears both the preview and the stored key.

## Whitespace cleanup

Print CSS was too generous. v1.7.2:

- `@page` margin: `0.5in` ‚Üí `0.35in`
- Section padding (`.rep-doc-head`, `.rep-floor-section`, `.rep-legend`, `.rep-doc-foot`): `12px 0` ‚Üí `4px 0`
- Section bottom margin: `12px` ‚Üí `6px`
- Floor title bottom margin: `14px` ‚Üí `8px`
- AP table cell padding: `7px 8px` ‚Üí `4px 6px`
- The doc-foot's top margin is now `0` so it doesn't orphan onto its own page.
- When a cover is present, the immediately-following floor section no longer force-breaks (the cover already broke). When there's no cover, the inline header + first floor stay together.

Net effect on the Carnation report: no more empty half-page under the AP table, and the antenna specs legend fits with the table instead of getting pushed to a new page.

Four Duplicates visual fixes from Dave's v1.8.17 review.

Suite `v1.8.17 → v1.8.18`. WD Cloud Manager `v3.27 → v3.28`.

## 1. Toolbar delete-all button truly centered and larger

Was off-center by ~69 px because the toolbar's left flank (search + × + All + selCount) is wider than the right flank (owner-toggle + New Site + Live + ⚙), and two flex spacers can only split leftover space equally — they can't overcome an asymmetric flanks.

Fix: `body.tool-cloud .toolbar { position: relative; }` plus `body.tool-cloud .dup-toolbar-btn { position: absolute; left: 50%; transform: translateX(-50%); }`. Absolute-positioned relative to the toolbar, dead-center regardless of flank width.

Also bumped: padding `5px 14px → 8px 24px`, font-size `12px → 14px`, box-shadow tinted amber so it reads as the primary page action.

Verified: `tbBtnOffsetPx: 0.0` from page center.

## 2. Cluster header — grey → purple

Dave: *"I'm not really loving the grey headers — it just [doesn't] seem like a header. Unfortunately I don't have an idea of what color to use but grey isn't it."*

Every other color on the Duplicates page is claimed:
- **Blue** — cloud rows
- **Green** — local rows
- **Amber** — extras + Duplicates tab
- **Teal** — Report page brand
- **Red** — Delete

**Purple** (`#8b5cf6`, `--purple-dark: #7c3aed`) is the only palette color not used elsewhere on this view. Now:

- Background: `rgba(139,92,246,0.14)` dark, `0.16` light
- Border-left: `4px solid var(--purple)`
- Title color: `var(--purple)` dark, `var(--purple-dark)` light
- Hover: tint bumps to 0.22 / 0.24

Reads immediately as "section start" — no more visual blend with the rows below it.

## 3. "What am I looking at?" banner centered

Was left-aligned. Now:
- `.dup-explain { text-align: center }`
- `.dup-explain-body { max-width: 900px; margin: auto }` so long lines don't stretch full width but still center within the banner.

## 4. "Normalized name" explained inline

Added a subtle italic hint right after the term:

> …share a **normalized name** *(punctuation, spacing, and case ignored — so `SNAN2-3030-Baseline` matches `SNAN2 3030 Baseline`)* AND have at least one extra copy…

No more mystery meat — the concept is defined the first time it appears.

## Bonus confirmation

Dave asked whether the button counts are dynamic. They are:
- Per-cluster: `Delete N extras (keep the pair)` — where N = unmatched items in that cluster.
- Toolbar: `Delete all N extras` — where N = total unmatched across every cluster.
- Both re-render on every `renderDuplicates()` call, so they update the instant clusters change (e.g. after a delete).

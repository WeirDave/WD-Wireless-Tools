Report tool rebuilt as a staged wizard, AP filter scales to 500+ APs with grouping and search, and the local server now uses Waitress so wireless admins downloading the suite no longer see Flask's scary "do not use in production" warning.

Suite `v1.8.33 → v1.8.34`. WD Report `v2.1` (workflow rebuild — same version tag, major UX overhaul).

## 1. Report rebuilt as a staged wizard

The old sidebar-with-everything layout was cluttered — Dave's words: *"the layout we currently have is cluttered. We need to make reporting a 'staged' process instead and treat it as a thorough product not a quick tool."*

New flow:

```
① DROP FILE  ➜  ② CHOOSE TEMPLATE  ➜  ③ CONFIGURE  ➜  ④ REVIEW & PRINT
```

Every stage is full-width. A stepper across the top shows the current position; users can jump back to earlier steps at any time.

**Stage 2 · Template picker** — Word-style template cards, each with:
- An inline-SVG page-mockup thumbnail (cover + floor plan + per-AP table)
- Title + one-line description
- "What's inside" checklist — 4 lines describing each section of the report
- "Best for:" callout — the use case in one line
- Big **Use this template** CTA

The BOM report renders as a dimmed "Coming soon" card so the platform reads as a suite-of-templates rather than a one-trick tool.

**Stage 3 · Configure** — three grouped cards replace the sidebar strip:
- **Cover page** — include cover toggle, logo upload/remove
- **Report options** — per-report checkboxes, each with a description line
- **APs to include** — full-width AP filter (see next section)

Two clear nav actions at the bottom: `← Change template` and `Generate report →`.

**Stage 4 · Review & Print** — the rendered report with a slim top bar (Back to Configure, Open another `.esx`, Print / Save PDF). Only this stage prints — the wizard chrome is hidden via print CSS.

## 2. AP filter scales to 500+ APs

Tested against a real 432-AP project. The old flat list of 432 checkboxes was unusable. New controls:

- **Group-by pills**: `[ None | Color | Floor | Building | Model ]` — click to reorganize the AP list live
- **Search box** — filter by AP name as you type
- **Collapsible groups** with `N of M` checked count on each header, plus a per-group **Toggle all** button
- **Auto-collapse** when the project has > 30 APs, expanded for smaller projects
- **Color grouping** shows the actual color swatch on each group header (Dave uses color in Ekahau for area-of-service coding: blue = office, red = warehouse, etc.)
- Every AP row shows both the name and the model designator (`Mist · AP47D`)

## 3. Antenna Placement options clarified

Each option in the report card now has a **bold label + description subtitle** so the effect is self-explanatory:

- **Floor plan overview page** — Every AP plotted on the plan with SVG direction arrows.
- **Antenna specs reference** — Final table listing every antenna model with gain and beam width.
- **Show mount heights in both units** — Meters primary, feet in parentheses — e.g. "2.5 m (8'2\")".
- **Show compass headings alongside azimuth** — Azimuth shown as "137° (SE)" instead of just "137°".
- **Include directional APs** (default ON) — Standard case — APs whose antennas have a specific azimuth.
- **Include omni APs** (default OFF) — APs with only omni antennas — rendered as "Omni — no direction". Off by default because this report focuses on directional mounting.
- **Flag APs with external antennas** (default ON) — Highlight APs that use external antennas so installers know they need separate mounting.

Options that can't apply to the current project auto-disable with a tooltip explaining why. Example: on all-integrated-antenna projects (like the Mist AP47D test file), "Antenna specs reference" grays out with *"No beam-width data in this project (all-integrated antennas)."*

## 4. Waitress replaces Flask's dev server

The old terminal output showed Flask's built-in dev-server warning:

```
* Serving Flask app 'server'
* Debug mode: off
WARNING: This is a development server. Do not use it in a production deployment.
* Running on http://127.0.0.1:8675
```

Confusing for someone downloading the tool for the first time — reads like the software is unfinished.

New output:

```
WD Wireless Tools  →  http://localhost:8675/
Ready. Press CTRL+C to quit.
```

Under the hood: switched to Waitress (production-grade pure-Python WSGI, ~200 KB) with a graceful fall-back to Flask's dev server if Waitress isn't installed. `waitress>=3.0` added to `requirements.txt` — auto-installed by `run.bat` / `run.command` on first run.

Hotfix for a runtime error in v1.8.15 that blocked the Duplicates tab from rendering.

Suite `v1.8.15 → v1.8.16`. WD Cloud Manager `v3.25 → v3.26`.

## The bug

`renderCluster` failed with `can't access lexical declaration 'sideCls' before initialization` — a classic JavaScript temporal-dead-zone error. In v1.8.15 I added `side-` + sideCls` to the `rowCls` array **before** the `const sideCls = …` declaration. `const` bindings are hoisted but not initialized until the actual declaration line, so reading the identifier earlier throws.

The Duplicates tab hit this on every render, showing "Load failed: can't access lexical declaration 'sideCls' before initialization" and no clusters.

## Fix

Moved the `const sideCls = it.side === 'cloud' ? 'cloud' : 'local'` declaration ABOVE the `rowCls` array in the `cl.items.forEach` block. Same value; correct order.

## Verified in browser

- Duplicates tab loads cleanly, 6 clusters render.
- 7 rows with `.side-cloud` (blue tint), 11 with `.side-local` (green tint).
- All 6 extras get the amber `box-shadow` outline as designed.
- `.dup-actions-strip` renders above the clusters with the count summary and Delete-all button.

## Files changed

- `web/assets/js/cloud.js` — `renderCluster` reorders local declarations inside the `forEach`: `sideCls` and `sideIcon` now precede `rowCls`.
- `web/cloud.html` — version chip `v3.25 → v3.26`.
- `web/pages/hosted-index.html` — suite chip `v1.8.15 → v1.8.16`.

# WD Scale UX fixes: 2 dp precision + drop echo rows

Two small papercuts from real use of v1.6.0 that came in the same afternoon.

## What changed

### Precision: everything is now `xx.xx`

Ekahau's scale field only accepts 2 decimal places, so any value we handed back with more was going to get chopped anyway. v1.6.0 rendered 4 dp for feet/meters and 3 dp for inches — technically more precise, in practice useless. v1.6.1 rounds every output to 2 dp and strips trailing zeros so you get `163.47` and not `163.4744` or `163.47000`.

The mirrored input on the opposite side (populated when you type on one side, showing the raw decimal on the other) now also uses 2 dp — so a copy-paste from that field can't sneak in extra precision either.

### Dropped the "echo" rows

The imperial panel had a **Feet & inches** output row, and the metric panel had a **Meters & mm** row. If you typed `536'4"` on the imperial side, the imperial panel's "Feet & inches" row proudly showed you `536' 4"` — literally your input, back at you. Same on the metric side.

Both rows are gone. Each panel now shows the two decimal outputs its neighbor's units *don't* provide:

- **Imperial panel** → decimal feet, total inches
- **Metric panel** → decimal meters, total millimeters

The other unit-family conversions live in the *opposite* panel (which is where you're looking anyway when converting), so no information is lost. Just less clutter.

## Files changed

- `web/scale.html` — removed `#outFtIn` and `#outMmm` rows; bumped page version chip to `v1.1`.
- `web/assets/js/scale.js` — precision `3–4 → 2` across all outputs and both mirror-input writes. Element list narrowed to the four remaining outputs.
- `web/pages/hosted-index.html` — version chip → `v1.6.1`.
- `README.md` — WD Scale sub-version → `v1.1`.

`formatFeetInches` and `formatMetersMM` remain on `window.WDScale` for anyone who imports the module externally — they just aren't rendered in the default UI anymore.

## Verification

1. Hard-refresh `/scale` (or the hosted `/scale/` once Pages redeploys).
2. Type `536'4"` on the imperial side — decimal feet reads `536.33`, total inches `6436`. Metric side reads `163.47 m` and `163474.4 mm` (mm keeps 2 dp of precision below the millimeter, so anything not a round mm stays readable).
3. Type `12m 500mm` on the metric side — imperial side reads `41.01 ft` and `492.13 in`.
4. Neither panel echoes your input.

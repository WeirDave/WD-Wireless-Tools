## Fix: /favicon.ico no longer 404s in server logs

v1.8.29 committed `web/favicon.ico` at the web/ root and wired every HTML page to the multi-size icons under `/assets/`. Browsers pick up the visible tab icon from the explicit `<link rel="icon">` tags, so nothing looked broken — but tools, bots, and RSS readers that hit `/favicon.ico` directly (by long-standing convention) still got a 404 because there was no Flask route serving `web/favicon.ico`.

Added a one-off route in `server.py`:

```python
@app.route("/favicon.ico")
def favicon():
    return send_from_directory(WEB, "favicon.ico", mimetype="image/vnd.microsoft.icon")
```

Verified: `/favicon.ico` returns 200, `Content-Type: image/vnd.microsoft.icon`, 8890 bytes — same file the HTML link tags reference at `/assets/wd-wireless-tools-v8.0-multi-size.ico`.

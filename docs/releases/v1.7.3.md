# WD Report v1.2.1 — File badge styling on the Report toolbar

You noticed the filename in the top-left of `/report` was showing as plain unstyled text — nothing like the `.mono` blue pill it renders as on Quick Walls.

Cause: the file-badge CSS was scoped to `.header .file-badge`. Quick Walls wraps its toolbar in `<div class="header">`, so it matched. Report wraps its toolbar in `<div class="report-toolbar">`, so it didn't — the badge got no styling.

Fix: broadened the two selectors to also match `.report-toolbar .file-badge` (and `:hover`). Now the filename renders as the same blue monospace pill on both tools.

## Files changed

- `web/assets/wd-tools.css` — selector extended, one-line change on each of the two rules.
- `web/pages/hosted-index.html` — version chip → `v1.7.3`.

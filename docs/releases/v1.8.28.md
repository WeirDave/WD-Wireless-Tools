A single-vocabulary pass on buttons across the whole suite. Every button — filled, outlined, small icon, folder, filename, theme toggle — now speaks the same language: pill shape, navy accent border, 3 D lift on hover, navy-blue text on hover. And Quick Walls' primary buttons drop the shouty blue fill for a lighter outlined treatment.

Suite `v1.8.27 → v1.8.28`. WD Cloud Manager `v3.38 → v3.39`.

## 1. A new `--nav-blue` accent token

`--nav-blue: #0668D9` added to the palette (same value in both themes). This is the accent blue used on outlined borders throughout the suite — theme-toggle, buttons, folder, filename, peek eye. Reused from the Duplicates cluster header work so the same "click me" cue is consistent across every clickable pill.

## 2. Every button gets a navy border and a centered label

- Base `.btn` now carries `border: 1 px solid var(--nav-blue)` — the shared accent shows through on outlined variants and defines the shape of filled variants.
- Filled variants (`.btn-primary`, `.btn-blue`, `.btn-green`, `.btn-red`, `.btn-amber`) get `border-color` matching their fill so the rim doesn't fight the fill.
- Added `justify-content: center` to base `.btn` so full-width buttons (like Quick Walls' Save the *.esx) center their label instead of anchoring left.
- Every `.btn` — every variant, every tool — transitions `transform`, `box-shadow`, `background`, and `border-color`, so the shared 3 D lift on hover is universal.

## 3. Quick Walls primary → outlined blue instead of solid fill

Apply and Save the *.esx were solid-blue filled and dominated the workspace. They're now outlined with `--nav-blue` — same primary role, quieter presence, still obviously actionable via the 3 D lift on hover. Disabled state uses `--text-3` / `--border`.

`body.tool-walls .btn:hover` also shifts the text to `--nav-blue` on hover — the same "text turns navy on hover" pattern the file-badge uses.

## 4. The theme toggle joins the club

- `.theme-toggle`'s border was `--border` (grey). Now `--nav-blue`, matching every other pill in the header.
- Hover: `transform: translateY(-2 px)` + `0 6px 18px rgba(0,0,0,.22)` shadow, plus text color to `--nav-blue`. Same 3 D lift as buttons and cards.
- `:active` returns to `y = 0` with a smaller "settled" shadow.

## 5. Filename → real clickable button

`.file-badge` (the loaded .esx / project filename in Walls and Report headers) was a subtle blue tint that people missed. It's now a proper pill button — mono font, navy border, `--surface-raised` background, 6 px 14 px padding, 3 D lift on hover, text turns `--nav-blue` on hover. Reads as "click me to open a different file," which is what it does.

## 6. Cloud Manager's Folder button was the outlier

`.settings-btn` (the "📁 Folder" pill in the Cloud toolbar) was still using the old 10 px / 3 px-radius / rgba-white treatment from before the unification. Now cloned exactly from `.theme-toggle`:

- Same `padding: 6 px 12 px 6 px 10 px`
- Same `font-size: 12 px`
- Same `gap: 7 px`
- Same navy border, radius, hover transition
- Icon wrapped in `<span class="ico">` and included in the shared `.ico` rule (14 px / line-height 1), so the folder emoji renders at the same size as the theme-toggle moon/sun

Verified: both pills now measure **28 px tall** with identical padding, font-size, gap, radius, and border.

## 7. Save the *.esx label

Renamed from "Save .esx" to "Save the *.esx" — clearer as an action button, and now centered inside the full-width pill thanks to base `.btn` gaining `justify-content: center`.

## 8. Row-action circles unified — 26 × 26 across every tool

Row action buttons were inconsistent between tools:

- Cloud Manager `.icon-btn`: 26 × 24, `border-radius: 3 px` (rounded square)
- Walls `.btn.btn-icon`: `50 %` radius but walls-specific `.btn` padding was stretching it into a 38 × 28 pill
- Walls `.btn.btn-sm` shortcut button (the `#` on wall-cards) was `.btn.btn-sm` with no `.btn-icon`, so it stayed pill-shaped while the ✏ / × next to it were circles
- Cloud Sites/Projects peek eye (`.src-badge`) was transparent + no border + asymmetric padding — a text-only accent, not a button

All fixed:

- `.icon-btn` → 26 × 26 / `50 %` / navy border / 3 D hover
- `.btn.btn-icon` → 26 × 26 (previously 28 × 28), `50 %` radius kept
- `body.tool-walls .btn.btn-icon` added — locks 26 × 26 padding 0 so walls padding can't stretch it back
- Walls' `#` shortcut button now `.btn.btn-icon.btn-sm` matching the ✏ / × next to it
- `.src-badge` → 26 × 26 / navy border / 3 D hover. `.hasrc` variant keeps amber (border + text) since it signals "source files not on cloud"

Every row-action icon button in the suite now measures **26 × 26 px** with a `50 %` border-radius. Three tools, one silhouette.

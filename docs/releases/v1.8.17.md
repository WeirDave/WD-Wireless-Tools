Three fixes from Dave's Duplicates review of v1.8.16.

Suite `v1.8.16 → v1.8.17`. WD Cloud Manager `v3.26 → v3.27`.

## 1. Row tints match Project Files exactly

Dave: *"if you look at Project Files where you've got the highlighting for local and for cloud on the rows for local and cloud sections, you're not using the same blue and green that you are for those pages."*

Fixed by lifting the exact tint values from `.glh-letter.cloud` / `.glh-letter.local` (the A-Z section-header chips on Project Files):

- `.dup-item.side-cloud` — `rgba(30,119,172,0.12)` (was `0.06` — half-strength).
- `.dup-item.side-local` — `rgba(95,171,79,0.14)` (was `0.06`).

Light mode gets a stronger tint since the previous `0.09` looked grey on white:

- Light cloud: `rgba(30,119,172,0.20)`
- Light local: `rgba(95,171,79,0.22)`

Hover states scale proportionally. The two pages now share one visual language.

## 2. Cluster action buttons all on the right

Previous layout:

```
[Keep newest] [Keep largest] [spacer] [count] [Delete checked]
```

Dave: *"that button shouldn't be on the left, it should be on the right with all the other buttons — we've got all the buttons on the right except for that button which is crazy."*

New layout:

```
[count] [spacer] [Keep newest / Delete N extras (keep the pair)] [Delete checked]
```

Count anchors the left, all destructive buttons grouped on the right. Cleaner and consistent with every other toolbar on the page.

## 3. "Delete all N extras" moved to the toolbar row

Was in its own action strip below the toolbar. Dave: *"when I wanted you to put the Delete all six extras it's supposed to be in the same row as the search bar just in the middle."*

Moved to the toolbar (`.toolbar` container) with two `<span class="spacer"></span>` sandwiching it — both spacers `flex: 1`, so the button lands in the middle of the flex row between the search-side elements and the owner-toggle/Live/settings on the right. Only rendered when the Duplicates tab is active AND there's at least one extra to clean up.

The old `.dup-actions-strip` inside `#rowsContainer` is gone.

## Files changed

- `web/assets/js/cloud.js` — cluster `.dup-actions-bar` layout reordered (count → spacer → keep/delete buttons → delete-checked). `renderDuplicates` no longer emits the actions strip; it updates the toolbar button in place.
- `web/cloud.html` — added `#dupDeleteAllToolbarBtn` in the toolbar between two spacers. Version chip `v3.26 → v3.27`.
- `web/assets/wd-tools.css` — `.dup-item.side-cloud` / `.side-local` tint values raised to match Project Files section-header values; light-mode variants strengthened; hover states scale. Removed `.dup-actions-strip` styling; added a compact `.dup-toolbar-btn` variant.
- `web/pages/hosted-index.html` — suite chip `v1.8.16 → v1.8.17`.

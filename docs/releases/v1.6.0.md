# WD Scale — new tool: feet/inches ↔ decimal ↔ metric

A Google-tab-killer for scale calibration. Every time an architectural drawing showed `536'4"` and Ekahau asked for a decimal, the workflow was: alt-tab → Google → "536 feet 4 inches to decimal" → squint at result → alt-tab back. Now: paste into WD Scale, copy the answer, done.

**Live in your browser**: [weirdave.github.io/WD-Wireless-Tools/scale](https://weirdave.github.io/WD-Wireless-Tools/scale/) — no install needed, same as Quick Walls.

## What it does

Two side-by-side panels — Imperial and Metric. Type on either side, both sides update live.

**Imperial input accepts:**

- `536'4"` — feet + inches
- `4' 6-1/2"` or `4' 6 1/2"` — architectural fractions
- `1/2"` — bare fraction
- `536.333'` — decimal feet
- `6436"` — inches only
- `536 ft 4 in` / `536 feet 4 inches` — word forms
- `536` — a bare number is interpreted as feet
- Fancy typographical primes (`′` `″`) work the same as `'` `"`

**Metric input accepts:**

- `12m 500mm` — meters + mm
- `163.475m` or `163.475` — decimal meters
- `163475mm` — millimeters only
- `1250cm` — centimeters
- European comma decimals (`163,475m`) — normalized to a dot before parsing

**Every input produces six outputs**, each with a copy button:

| Imperial side | Metric side |
| --- | --- |
| Decimal feet | Decimal meters |
| Total inches | Total millimeters |
| Feet & inches (nearest 1/16") | Meters & mm |

## How the parsers work

Two separate regex pipelines, one per unit family. Both accept a **bare number** as a convenience (feet on the imperial side, meters on the metric — matches how surveyors and architects actually speak). Any other input needs a unit token.

The imperial parser handles the tricky architectural-fraction case with three layered regexes tried in order:

1. Mixed number with fraction: `4-1/2"`, `4 1/2"`, `6 1/2 in`
2. Bare fraction: `1/2"`, `3/8`
3. Plain decimal or whole inches: `4.5"`, `4 in`, `4 inches`

The metric parser matches `mm` and `cm` **before** `m` so the shorter token doesn't get greedily consumed. All three run against the same string; the leftover after each substitution feeds the next.

Feet-and-inches output rounds to the nearest 1/16" using an integer-sixteenths approach so rollover (e.g. `15.99/16` → `1` inch) is arithmetic, not conditional. Fractions are reduced with a `gcd` so you get `1/2` instead of `8/16`.

Tested (via a Python port of the parser) against 20+ input forms — every case round-trips: `30.48m` in the metric panel produces exactly `100'` on the imperial side.

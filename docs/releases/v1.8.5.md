# v1.8.5 — Section-letter colors + light-mode contrast fix

Two small visual fixes on top of v1.8.4:

- **A-Z section letters now match their column colors**: cloud letter is `--blue` (`#1e77ac`) matching the "Cloud Projects" column header; local letter is `--green` (`#5fab4f`) matching "Local .esx". Removes the teal outlier and keeps the color language consistent across the ledger.
- **Light-mode `--text-3` and `--text-2` darkened site-wide**. The old `#9ca3af` was 2.5:1 against white — fails WCAG AA (4.5:1 required). New values pass:
  - `--text-2`: `#6b7280` → `#4b5563` — **7.56:1** (AAA)
  - `--text-3`: `#9ca3af` → `#6b7280` — **4.83:1** (AA)
  - Same shift applied to `.tool-walls` and `.tool-home` light overrides.

Suite `v1.8.4 → v1.8.5`. WD Cloud Manager `v3.14 → v3.15`.

## Section-header color mapping

Cloud column and local column already have blue/green identity via `.lh-cell.cloud` / `.lh-cell.local`. The A-Z sub-headers now carry the same coding — cloud letter on a subtle blue tint, local letter on a subtle green tint. Same 18 px / 900 weight / 3 px letter-spacing sizing from v1.8.4. Same solid opaque background so sticky doesn't look translucent. The teal outline / accent from the previous cut is gone.

Light-mode letter chip tints are slightly softer (10% blue, 12% green) than the dark-mode tints (12% blue, 14% green) to avoid over-saturating on a white page.

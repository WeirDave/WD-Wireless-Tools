Default port changed from `8765` to **`8675`** — half a Tommy Tutone tribute (`867-5309`, Jenny), half a fresh port less likely to collide with anything a corporate desktop might already be listening on.

Suite `v1.8.32 → v1.8.33`. No other user-visible changes.

## What changed

`server.py` now defaults to port **8675**. The URL you land on is `http://localhost:8675/`. Everything else — the launcher scripts, browser auto-open, API routes — is identical.

The `PORT` env-var override still works if you need a different port:

```bash
# Windows
set PORT=8676 && run.bat

# macOS
PORT=8676 bash run.command
```

## Why the change

Nobody uses this suite yet — this is Andrew's onboarding week — so no one has 8765 memorized. If we're going to pick a port, we might as well pick one that sticks in your head. `8675` is the first four digits of `867-5309`. Also lower collision surface than 8765, which shares its neighborhood with a bunch of dev-server defaults (8080, 8443, 8888).

## Files changed

- `server.py` — `PORT` default `8765` → `8675`.
- `README.md` — every `localhost:8765` reference updated (Quick Start, First Run, Troubleshooting).

- `PROJECT_MEMORY.md`, `.claude/launch.json` — dev references updated for consistency.

Historical release notes (like `v1.8.1.md`) that mention 8765 are left alone — they document what was true at that time.

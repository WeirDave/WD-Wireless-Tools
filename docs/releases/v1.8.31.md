Favicon now reacts to the **browser's** color scheme, not the app's theme toggle. White favicon in dark-mode browsers, color favicon in light-mode browsers — automatic, no JS.

Suite `v1.8.30 → v1.8.31`. No other user-visible changes.

## What was wrong

v1.8.29 shipped `WD.syncFavicon()` — a JS function that rewrites every icon link's `href` on page load based on the **app's** `data-theme` attribute. That's the wrong signal: `data-theme` comes from the app's own light/dark toggle button, stored in `localStorage`, and has nothing to do with whether the user's browser (or OS) is running in dark mode.

Result: if you had the browser in dark mode but the app in light mode (or the reverse), the favicon didn't match the browser's chrome.

## Fix

Every `<link rel="icon">` tag now has a `media="(prefers-color-scheme: light|dark)"` variant. Browsers pick the matching icon at parse time based on the OS/browser color scheme:

```html
<link rel="icon" media="(prefers-color-scheme: light)"
      href=".../wd-wireless-tools-v8.0-multi-size.ico" sizes="any">
<link rel="icon" media="(prefers-color-scheme: dark)"
      href=".../wd-wireless-tools-v8.0-white-multi-size.ico" sizes="any">
<link rel="icon" type="image/png" sizes="32x32" media="(prefers-color-scheme: light)"
      href=".../wd-wireless-tools-v8.0-32x32.png">
<link rel="icon" type="image/png" sizes="32x32" media="(prefers-color-scheme: dark)"
      href=".../wd-wireless-tools-v8.0-white-32x32.png">
<link rel="apple-touch-icon" sizes="180x180" href=".../wd-wireless-tools-v8.0-180x180.png">
```

Applied to all 12 HTML pages via `scripts/add_favicon_media.py`. `apple-touch-icon` is left alone — Apple ignores `media` there.

`WD.syncFavicon()` is now a no-op stub (kept for API compatibility so no other callers break). Media queries do the work, live-update on system-theme change, and don't fight browser icon caching.

Verified in-browser via `preview_resize` emulation:
- `prefers-color-scheme: dark` → only white-variant links match
- `prefers-color-scheme: light` → only color-variant links match

You asked for a **loader for multiple styles of reporting**. WD Report is now built around a report registry — adding a new report type is one entry in `REPORTS = { ... }`. First two reports ship today:

- **Directional Antenna Installation** (`docName: "AP Placement"`) — same as before, refactored into the registry
- **Bill of Materials** (`docName: "Bill of Materials"`) — new

## The registry

Each report is a plain object:

```js
{
  id, label, description,   // shown in the sidebar picker
  docName, coverBrand,      // drives filename + header text
  sidebar: [{ id, label, default }],   // per-report checkboxes
  render: function (aps, opts, ctx) { return html; },
}
```

- The **sidebar picker** is now generated from the registry — one radio per report, with the description as a muted sub-line. No hardcoded HTML.
- **Per-report options** render into a dynamic slot on the sidebar. Switching reports swaps the whole options block. Cover + logo + AP filter are common controls above the slot.
- The **filename convention** `Report - {docName} - {Site Name}` is driven by the current report's `docName`. Picking BOM changes the browser tab title and the Save-as-PDF suggested filename to `Report - Bill of Materials - {Site}`.
- Each renderer gets a `ctx` object with shared helpers (`primaryRadio`, `compass`, `metersToFt`, `fmt`, `floorPlanForAp`, `cover`, `inlineHeader`) so future reports don't re-implement them.

Adding a third report is: define a `render(aps, opts, ctx)` function, drop an entry into `REPORTS`. Sidebar picker + dynamic options wire themselves up.

## Bill of Materials

Two clean tables, no floor plans:

- **Access Points** — grouped by `vendor + model` with a Qty column and a Total row. Sorted by quantity desc.
- **Antennas** — grouped by antenna type `name`. Each AP contributes its distinct antennas (not each radio) so a dual-band AP with two different antennas counts once per antenna, not once per radio. Sidebar option **"External antennas only"** filters to `apCoupling` containing `EXTERNAL` — useful for procurement lists when you only need to order antennas that aren't built into the AP.

For the Carnation project (7 AP-677s, all internal antennas), the default view shows 7 APs and 28 antenna instances (4 per AP: 2.4/5/6/BLE). Toggling **External antennas only** yields an empty antennas table — semantically correct: the AP-677 ships with everything integrated, nothing to order separately.

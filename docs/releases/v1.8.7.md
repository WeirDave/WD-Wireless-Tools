# v1.8.7 — Settings gear visibility

*"you can't tell it's there right now, it is easily overlooked"* — Dave, on the ⚙ Settings button in light mode.

The `.btn-icon-only` from v1.8.6 was too subtle: transparent background + light `--border` on a white page = practically invisible. Fixed by inverting the fill.

Suite `v1.8.6 → v1.8.7`. WD Cloud Manager `v3.16 → v3.17`.

## The change

`.btn-icon-only` now uses `background: var(--text)` and `color: var(--surface)` — an intentional color inversion:

| Theme | Background | Icon |
| --- | --- | --- |
| Light | `#111827` (near-black) | `#ffffff` (white) |
| Dark | `#e7ebf0` (near-white) | `#18212c` (near-black) |

The inversion works because both `--text` and `--surface` are already theme-scoped tokens. Same rule fires in both modes, opposite result.

Reads as "utility" because it has no hue — distinct from `.btn-blue` (primary action), the outlined `.btn-secondary` (secondary action), the green `#liveBtn.live-on` (toggle-on), and the purple `.owner-btn.active` (filter). No two elements share a color language on the Cloud toolbar now.

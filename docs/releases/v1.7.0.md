# WD Report — installer-facing PDF reports from your .esx

Ekahau's UI hides the antenna-mounting info that a physical installer actually needs — direction, tilt, mount type, height. That data lives in the `.esx`; it just isn't surfaced anywhere printable. **WD Report** parses the `.esx` in the browser and turns it into a print-ready handoff document.

**Live in your browser**: [weirdave.github.io/WD-Wireless-Tools/report](https://weirdave.github.io/WD-Wireless-Tools/report/) — same treatment as Quick Walls and Scale, no install.

## First report: Directional Antenna Installation

Drop an `.esx`, get a document with:

1. **Header** — project name, AP count, floor-plan count, generation date.
2. **Per floor** — the floor-plan bitmap with an SVG marker + direction arrow on every AP.
3. **AP table** — one row per AP: name, vendor, model, mount type, mounting height (meters, feet in a muted second line), azimuth (degrees + compass abbreviation like `NNW` / `ESE`), tilt, antenna model. Values come directly from `simulatedRadios.json` → `antennaDirection` / `antennaTilt` / `antennaHeight` / `antennaMounting`.
4. **Antenna specs** — distinct antennas used, with frequency band, gain, and beam widths from `antennaTypes.json`.

Verified against a real project (CRNT1 Carnation Farms — 7 Aruba AP-677s outdoor directionals): every azimuth, tilt, mount height, and antenna model came through correctly. AP-001 wall-mount at 10 ft aimed 313.2° (NW), AP-006 wall-mount at 10 ft aimed 88.6° (E), etc.

## How it works

Every `.esx` is a ZIP of JSON files. We read `accessPoints.json`, `simulatedRadios.json`, `antennaTypes.json`, `floorPlans.json`, and `images.json`, join them by ID, and render a document. Bitmap floor plans are pulled out as blobs and re-tagged with the correct MIME (JPEG/PNG/SVG) so a plain `<img>` can render them.

The overlay uses SVG with a `viewBox` set to the floor plan's logical dimensions (`width` × `height` from `floorPlans.json`), so AP coordinates from `location.coord` plot directly without any math — SVG scales the overlay to match the image below it. Direction arrows are drawn as a small triangle rotated by the antenna's `antennaDirection` degree value (0° = North, Y-down convention).

Ceiling-mount APs get a purple dot with no arrow (they're omni by convention). Wall/pole-mount APs get a teal dot with a triangular direction cone pointing at their azimuth.

Sidebar options are all live (no re-drop needed):

- **Include floor plan overview** — toggle the big map
- **Antenna specs** — the legend section at the bottom
- **Feet alongside meters** — muted secondary units under the metric primary
- **Compass abbreviations** — `NNW` in a muted span next to `342.6°`
- **Per-AP checkboxes** — hide APs you don't want in the printout

**Print CSS**: `@page` margin, sidebar/toolbar hidden with `.noprint`, backgrounds forced white, each floor section starts on its own page (`page-break-before: always`). Browser → Ctrl-P → Save as PDF gives you a clean handoff without any extra tooling.

## Verification

1. Restart the Flask server (new `/report` route).
2. Hard-refresh (`Ctrl+Shift+R`).
3. Open `/report`. Drop your Carnation `.esx` (`CRNT1 - 30103 NE Carnation Farm Rd, ... - EXT PD.esx`).
4. Overview shows the floor plan with 7 teal markers, each with an arrow at its `antennaDirection`.
5. Table below shows every AP, mount, height, azimuth (with compass), tilt, antenna. Toggle the sidebar options — the report re-renders live.
6. Ctrl-P → Save as PDF → the sidebar and toolbar vanish, the document prints on white with page breaks between floors.

Bulk actions in Cloud Manager finally behave. Individual checkbox clicks now light up the toolbar (they used to only respond to "Select All"), and a new **Move to site…** button handles bulk-relocation of both cloud projects and local `.esx` files in one modal.

Suite `v1.8.37 → v1.8.38`. WD Cloud Manager `v3.41 → v3.42`.

## 1. Cloud Manager · individual checkbox clicks now update the toolbar

The bug: `onRowChkClick` was defined but never wired up. Clicking a single row checkbox toggled the DOM state visually, but `updateBulkBar()` was never called, so the Sync / Delete / Compare buttons stayed hidden. The only way to reveal them was to hit the "All" checkbox at the top of the toolbar (whose `onchange` handler *did* call `updateBulkBar`).

Fix: bind `onRowChkClick` as a delegated click listener on `#rowsContainer`. Delegation survives every ledger re-render, so the wiring never goes stale.

## 2. Cloud Manager · bulk "Move to site…"

Previously the toolbar had **Sync →**, **Sync ←**, **Compare**, and **Delete selected**. There was no way to bulk-move a stack of projects into a site — you had to click the per-row `↪` icon on each cloud row individually, and local `.esx` files had no move option at all.

New **Move to site…** button appears on the Project Files tab. Handles a mixed selection:

- **Cloud-only projects** → assigned to the picked cloud site via `assign_to_site` (same Ekahau API as the per-row icon).
- **Local-only `.esx` files** → moved on disk into the site's local folder. If the folder doesn't exist yet, it's created inside your `output_dir`.

The site picker now lists **all** sites — matched pairs, cloud-only sites, AND local-only folders — with a `(cloud only)` / `(local only)` tag so you know which side exists. Picking a local-only site with cloud items selected fails cleanly with a toast ("no cloud counterpart") instead of silent breakage. `+ Create new site…` still works and only hits the Ekahau API when the selection actually contains cloud items.

## 3. Cloud Manager · toolbar buttons stay visible

The Sync / Compare / Move / Delete buttons used to appear and disappear based on what was selected. This made the toolbar feel unstable — click a checkbox, three buttons pop in; uncheck, they vanish. And when the wiring above was broken it looked like the buttons plain didn't exist.

Now they're always visible on the tab they belong to (Compare stays Sites-only, Move stays Projects-only). Each button validates its own selection on click and toasts a helpful message if there's nothing to act on:

- **Sync →** with no matched rows → "Select some matched rows first"
- **Delete selected** with only paired rows → "Bulk delete only applies to cloud-only or local-only rows"
- **Move to site…** with nothing selected → "Select cloud projects or local .esx files first"

# WD Report v1.1 — filename convention: `Report - AP Placement - {Site}`

You generated your first WD Report PDF and pointed out the naming pattern should be `Report - AP Placement - {Site Name}` — matching how Ekahau names its own AP Placement PDFs. This patch adopts that.

## What changed

**Site name extraction.** The `.esx` filename gets stripped of its `.esx` extension and any trailing " - {short project suffix}" (like `EXT PD`, `Baseline Survey`, `Cleanroom PD`, `Remediation`). The heuristic only strips if the trailing segment is under 30 characters AND contains no commas — so an address part like `WA 98014` is preserved, but a project qualifier `EXT PD` is dropped.

Given `CRNT1 - 30103 NE Carnation Farm Rd, Carnation, WA 98014 - EXT PD.esx`, the site name becomes `CRNT1 - 30103 NE Carnation Farm Rd, Carnation, WA 98014`.

**Document title (drives Save-as-PDF filename).** When you drop an `.esx`, the browser tab's `<title>` now becomes `Report - AP Placement - {Site Name}`. Chrome/Edge/Firefox use that string as the default filename in the Print → Save-as-PDF dialog, so hitting Save produces exactly `Report - AP Placement - {Site}.pdf` — no rename needed.

**Report header.** The h1 on the report itself shows the site name, and the small brand line above it now reads "Report · AP Placement" (was "Directional Antenna Installation Report"). Matches the filename convention exactly so the header, the tab title, and the saved PDF all line up.

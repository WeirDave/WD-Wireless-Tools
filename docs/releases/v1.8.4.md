## Local owner extraction

- `_esx_author(path, mtime)` opens the .esx zip, reads only `project.json`, extracts `project.history.createdBy`, lowercases and returns it. Empty string on any error (unreadable zip, missing key, etc.).
- Cached in a module-level dict `_ESX_AUTHOR_CACHE = {path: (mtime, author)}`. First scan of your ~150 .esx takes ~700 ms; every scan after that is ~0 ms until a file's mtime changes (you re-save it).
- `get_local_esx_files()` calls it inline. `build_projects_data` forwards `owner` on each local dict. The frontend gets it on every local row.

## Two owner formats — surfaced honestly

Your .esx files carry the author in **three** flavors:
- 86 files: `dpaine@anduril.com` (email)
- 26 files: `atran@anduril.com` (email — Andrew's)
- 17 files: `r david paine iii` (display name — the older ones)

The filter logic reflects this:
- Email string that isn't yours → **Others**
- Empty owner, your email, or any non-email string (display name) → **Mine**

Rationale: local display names can't cross-reference the cloud API (which returns emails), and any .esx sitting on your disk that says `"r david paine iii"` was almost certainly yours. Treating display names as "mine" gets the right result for 100% of your files today. If we ever add a Preferences map for "these display names are mine" it can override this.

## Toolbar three-way toggle

- New `.owner-toggle` component in the toolbar between the bulk-action buttons and `+ New Site`. Three buttons: `All` / `Mine` / `Others`. Active one gets a teal fill; others are ghost-styled.
- Only shown on the **Project Files** tab (Sites don't have per-file owners; hidden via `switchTab`).
- Selection persists to `localStorage` under `wd-owner-filter`. On page load, `syncOwnerToggle()` restores the highlight before the first `refreshData()`.
- The old `setowner` radio group in the Settings modal is gone — one source of truth.

## Cross-side filter semantics

A row is "Others" if **either** side (cloud OR local) carries someone else's email. So:
- Andrew's cloud file paired with your local copy → **Others** view (matches `atran@anduril.com` on the cloud side)
- Your cloud file paired with Andrew's local copy (rare but possible) → **Others** view too
- Both sides yours → **Mine** view

The old logic only looked at the cloud side; a cross-owner pair (your local + someone else's cloud) would have leaked into "Mine". Fixed.

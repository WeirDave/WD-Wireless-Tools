## Header and info panel now render pixel-identical

Dave was right — even though v1.8.21 gave both `.dup-head` and `.dup-explain` the same `rgba(217,119,6,0.08)` amber wash, they came out visibly different. Why: transparent color composites over whatever's underneath, and the two elements sit on different underlying surfaces:

- `.dup-head` is inside `.dup-cluster { background: var(--surface); }` — so it composites over `--surface`.
- `.dup-explain` sits directly in `#rowsContainer` on the app screen background — different underlying color.

Same 8% amber → different final rendered color.

Fix: switch both to the existing solid palette token `var(--amber-bg)` (`#2a2211` dark / `#fffbeb` light). Solid color = same rendered value regardless of what's beneath.

Verified in-browser:

| Theme | Header background | Info panel background | Match |
| --- | --- | --- | --- |
| Dark | `rgb(42, 34, 17)` | `rgb(42, 34, 17)` | ✓ |
| Light | `rgb(255, 251, 235)` | `rgb(255, 251, 235)` | ✓ |

Within each cluster, **faded** items are already paired; **amber-outlined** items are the extras worth deleting or merging.

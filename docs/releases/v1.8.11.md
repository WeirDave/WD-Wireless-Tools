# v1.8.11 — Duplicates: fade the pairs, bold the extras, bulk across clusters

Two things Dave asked for once he started using the Duplicates view:

1. *"the ones that match should be faded and the one that doesn't match should be bolded or something"* — for a 3-item cluster (say 1 cloud + 2 local), one local is usually the cloud's normal pair and the other is the actual duplicate. Without a visual cue, you have to eyeball which is which.
2. *"I checked one of the third options and then scrolled down and checked another third option thinking that I can hit delete checked and they would delete both, but it doesn't work that way"* — each cluster's Delete-checked button only saw its own cluster. No way to work across clusters.

Suite `v1.8.10 → v1.8.11`. WD Cloud Manager `v3.20 → v3.21`.

## Fade matched, bold extras

Each `.dup-item` now gets one of two classes based on the backend's `matched` flag:

- **`.is-matched`** — this file is already paired in the Sites/Projects view. Rendered at `opacity: 0.55` for name/location, `0.65` for size/date. Reads as "context, not the target."
- **`.is-extra`** — this file has no pair. Rendered with `font-weight: 700` on the name, a subtle `rgba(217,119,6,0.05)` amber tint on the row, and the side icon dyed amber. Reads as "this is what you're looking at — probably delete or merge."

Explanation banner updated to name the pattern: *"Items faded in each cluster are already paired in Sites/Projects — the bold ones are the extras worth investigating."*

Concrete effect on Dave's account: 212 dup items across 103 clusters — 206 faded, **6 bold**. Those 6 are the actual work to do.

## Cross-cluster bulk delete

New sticky bar appears at the top of the Duplicates view whenever any `.dup-item-check` is checked, anywhere in the DOM:

```
[☐ Select all across clusters]  ·  N selected across M clusters  ·  [Delete checked]
```

- **Per-cluster** "Delete checked" buttons still work (they've always worked *within* a cluster) — nothing removed.
- **New "Select all across clusters"** toggles every visible `.dup-item-check` in one shot. Uncheck one and the master flips back to indeterminate.
- **New "Delete checked"** in the top bar walks every cluster, collects checked items, and calls the same `_bulkDeleteItems` used elsewhere. One confirm dialog for the whole set.
- Sticky positioning (`top: 4px`) so as you scroll through clusters, the delete button stays accessible.

## Implementation

- `renderCluster` — adds `.is-matched` if `it.matched`, else `.is-extra`. Same check that already drives the "matched" pill on cluster items; just extended to the row class list.
- `renderDuplicates` — emits the `#dupBulkBar` sub-toolbar inside the view, above the cluster container. Calls `updateDupBulkBar()` after render.
- `dupChkChanged` — now also calls `updateDupBulkBar()` so checking a per-cluster box updates the global count.
- New `updateDupBulkBar()`, `dupBulkSelectAll(on)`, `dupBulkDelete()` — the last one iterates every `.dup-cluster`, normalizes each row's `data-iid` (same trick as the per-row delete fix in v1.8.10), and passes the collected items to `_bulkDeleteItems`.

Three coordinated changes so you can see at a glance whose files are whose in a 100+ project list:

- **Owner in parentheses** next to every cloud filename — dim grey when it's yours, orange when it's someone else's
- **"Show cloud projects by owner" setting** with three modes: All / Only files I created / Only files created by others
- **A-Z section headers** in the ledger — slim teal divider bars between letter groups so a long list reads as chunks, not a wall of rows

Suite `v1.8.2 → v1.8.3`. WD Cloud Manager `v3.12 → v3.13`.

## Ownership data flow

The Ekahau API exposes `history.createdBy` and `history.modifiedBy` on every cloud project (same field surfaced as "Owner" in the Ekahau Cloud Share dialog). The dataset endpoint has no separate `/permissions` route — the history block is the canonical source.

- `build_projects_data()` — cloud project dicts now carry `owner` (from `history.createdBy`) and `modifiedBy` (from `history.modifiedBy`), lowercased and stripped.
- `CloudManager.get_data()` — response now includes a top-level `currentUser` field (from the connected session's `ownerEmail`), so the frontend can classify each row as mine / not-mine without a second round-trip.

Local `.esx` files carry the same field internally at `project.history.createdBy`, but Ekahau writes it as a **display name locally** (e.g. `"R David Paine III"`) while the cloud API returns an **email** (`dpaine@anduril.com`). That mismatch means we can't cross-reference them for matching without a name↔email map, so we don't try. The cloud-side email is what powers the filter and badge.

## Ownership filter UI

- Settings modal gained a new radio group: **Show cloud projects by owner** — `All files` / `Only files I created` / `Only files created by others`. Persists to `localStorage` (`wd-owner-filter`).
- `renderLedger` gained a `passOwner(row)` gate. Local-only rows always pass (they're on your machine so they're yours by construction); cloud rows without an owner field also pass (fail-open). Only cloud rows with an owner email get filtered.
- The filter is scoped to the **Project Files** tab. Sites don't have per-file owners in the same way.

## Owner badge

`cloudCell` appends a `.owner-tag` span after the filename showing the owner's email in parens. When `owner !== currentUser`, the tag gets the `.other` class — an orange accent so unfamiliar owners stand out at a glance without needing to open the filter.

## A-Z section headers

- `renderLedger` inserts a `.ledger-group-head` divider row every time the first letter of the sort key changes.
- Non-alpha starts (numbers, symbols) group under `#`.
- Stripe alternation resets at each section header so the striping stays legible after the divider.
- The header is sticky (`top: 36px`, below the main ledger head) so you always see the current letter as you scroll.
- Slim teal gradient bar, uppercase, letter-spaced. Reads as a divider, not a data row.

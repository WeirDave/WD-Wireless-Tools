Three fixes from Dave's v1.8.23 review.

Suite `v1.8.23 → v1.8.24`. WD Cloud Manager `v3.33 → v3.34`.

## 1. Back to the translucent amber wash Dave liked

Dave: *"I like the orange 2 versions back, this solid is no good."*

v1.8.21 used `rgba(217,119,6,0.08)` — a translucent amber wash over whatever's underneath. Dave liked the way it read as orange. The problem then was that the two panels (`.dup-head` and `.dup-explain`) sat on different underlying surfaces, so the same rgba composited to different final colors. I "fixed" that with a solid color (`--dup-panel: #3a2510`/`#fde4bd`) but the solid didn't have the orange feel.

New approach: layered background — `linear-gradient(rgba(217,119,6,0.10), rgba(217,119,6,0.10)), var(--surface)`. The gradient paints a translucent amber wash ON TOP of an explicit `--surface` base. Both `.dup-head` and `.dup-explain` share the same base and the same overlay, so they composite pixel-identical.

Verified: base color `rgb(24, 33, 44)` (`--surface` dark) / `rgb(255, 255, 255)` (light) on both elements. The amber gradient overlay renders the same way on top.

## 2. Toolbar button fits inside the toolbar height now

The v1.8.21 delete-all button was `padding: 8px 24px` + `font-size: 14px` — ~30 px tall in a compact toolbar that's ~28 px tall. Absolute positioning centered it geometrically but the button visually overflowed the toolbar top and bottom.

Fix: shrunk button to `padding: 5px 18px` + `font-size: 13px` + `line-height: 1.2`. Result: 25.6 px button height inside a 37 px toolbar, `btnFitsInsideToolbar: true`. Vertically centered with proper breathing room.

## 3. Extras highlighting restored

Dave: *"the highlighting is broken now too."* When rows switched to a plain `--surface` background in v1.8.23, the extras' 2 px amber inset outline was the only signal — not strong enough against the white row.

Now extras get:
- **3 px amber inset outline** (was 2 px).
- **Subtle amber wash background** — `linear-gradient(rgba(217,119,6,0.10)…), var(--surface)`. Same trick as the panels: a translucent amber layer on top of the surface base, so it composites consistently.
- Bold name text and amber side icon stay.

Result: extras clearly pop against the matched (plain-white) rows around them, without going back to the "muddy mess" of a solid amber background.

The local-side section letter (A, B, C…) was right-aligned inside its chip. On a wide screen the letter sat far to the right, past the end of the file names below it — the eye scanning down the local column would land on the beginning of each file row, not the end. So the letter effectively wasn't there.

Fix: dropped `text-align: right` from `.glh-letter.local`. Both letters now use the default `text-align: start` — flush left in their column, right where the file names below them start.

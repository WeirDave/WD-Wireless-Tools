# v1.8.9 — Owner-toggle scoping + Duplicates tab polish

Four fixes from Dave's follow-up review:

1. *"the owner button of course does 'nothing' on the site tab which is slightly confusing"* — it was still visible on the Sites tab even though it only affects Projects. Now hidden until you switch to Project Files.
2. *"the tab color for duplicates when you are selected on it doesn't 'color up' like the other two"* — added amber active state, matching the existing blue (Sites) / green (Projects) color language.
3. *"I don't understand the duplicates tab yet"* — renamed to **Duplicate Projects**, added an inline explanation banner at the top of the view.
4. *"the icon for taking me to this tab on the project page in the row for a project is kind of hard to see in light mode"* — the ≈ hint badge now uses the proper amber palette color with a light-mode override for stronger contrast.

Suite `v1.8.8 → v1.8.9`. WD Cloud Manager `v3.18 → v3.19`.

## 1. Owner toggle hides outside Projects tab

The `.owner-toggle` was set to `display: inline-flex` (visible) by default in the HTML. On initial load the app starts on the Sites tab and never called `switchTab`, so the toggle stayed visible even though it has no effect there. Fix:

- HTML default: `style="display:none"` — hidden until proven relevant.
- `goToDashboard()` explicitly sets visibility based on `currentTab` on initial load.
- `switchTab()` continues to toggle visibility on tab switches (unchanged behavior).

## 2. Duplicates tab active color

Added `#tabDuplicates.active { background: var(--amber); border-top-color: #fde3b4; }` plus the resting-state top-border tint `rgba(217,119,6,0.55)`. Amber was chosen because:

- It's the third distinct color after blue (Sites) and green (Projects) — completes the tab color language.
- Amber has a "watch out" semantic that fits "these files might be duplicates."
- Not used elsewhere as a fill — no confusion with the small `.other`-owner amber tag.

## 3. Duplicates tab renamed + explanation banner

Tab label: **Duplicate Projects** (was "Duplicates"). The tooltip explains further: "Files with the same normalized name — spot accidental copies."

New `.dup-explain` banner renders at the top of the Duplicates view:

> **What am I looking at?**
> Files (cloud projects or local `.esx`) that share a normalized name with at least one other file. Punctuation, spacing, and case are ignored when comparing — so `SNAN2-3030-Baseline` clusters with `SNAN2 3030 Baseline`. Use this to spot accidental duplicates you can consolidate.
>
> **Mixed** — same name exists on both cloud and local (usually a normal pair — safe to ignore unless the sizes differ) · **Local only** — same file saved in multiple local folders (real duplicate — merge or delete one) · **Cloud only** — same project uploaded to Ekahau more than once (real duplicate — pick one and delete the extras)

Amber accent bar on the banner mirrors the tab color, so the whole Duplicates experience reads as one visual language.

## 4. ≈ badge contrast

The old `.dup-hint` used `#f0b85f` — 2.6:1 on white, well below WCAG AA. Fixed by:

- Dark mode: `color: var(--amber)` (`#d97706`) on `rgba(217,119,6,0.20)` background — 5.0:1 on `--surface`.
- Light mode: `color: var(--amber-dark)` (`#b45309`) on `rgba(217,119,6,0.15)` — 6.3:1 on white, AAA territory.
- Font size bumped 12 → 13 px so the glyph reads at a glance.

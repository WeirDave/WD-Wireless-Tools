# v1.8.6 — Button hierarchy pass

Two rounds of "styling issues" feedback from Dave:

1. *"the owner toggle buttons should NOT be the same colors as it is confusing as hell!"*
2. *"the setup button shouldn't be the same either — all those buttons need styling to be distinguishable from each other, what is a button what isn't etc"*

Fix: give every button on the Cloud toolbar a clearly-distinct visual role. Also fix the Report page where `.btn-blue` (Print / Save PDF) was rendering as a muted translucent button because a broad `.header-actions .btn` rule was overwriting its color.

Suite `v1.8.5 → v1.8.6`. WD Cloud Manager `v3.15 → v3.16`.

## Cloud toolbar button roles

Every action now has a distinct visual language:

| Element | Role | Style |
| --- | --- | --- |
| **+ New Site** | Primary action — creates something | Solid `--blue` fill |
| **Refresh** | Secondary action — frequent, not primary | Outlined transparent, `--text` label |
| **Live** | Toggle — has on/off state | Off: outlined + grey dot. On: solid `--green` fill + white dot with glow |
| **⚙ Settings** | Utility — configure preferences | New `.btn-icon-only` — 28 px square, transparent, only the icon |
| **Owner: All / Mine / Others** | Filter segmented control | `--purple` pill (from v1.8.5) |

The Live button now reads as a switch, not another action. The gear reads as a tool icon, not a call-to-action. Nothing on the toolbar competes with anything else.

## Report toolbar fix

`.header-actions .btn` was applying a translucent-white style to *all* buttons in the report top-bar, including `.btn-blue`. Result: "Print / Save PDF" looked identical to "Open another…" — you circled this earlier.

Scoped the translucent style to only `.btn-secondary`. `.btn-blue` in the report header now shows solid blue like everywhere else. Both light and dark theme overrides included.

## Segmented owner control (from v1.8.5, restyled here)

- Container: `--surface-raised` background, 1 px `--border`
- Inactive buttons: transparent + `--text-2` label
- Active button: `--purple` fill + white text + shadow
- Hover on inactive: `--surface-hover` background

Purple `#8b5cf6` is unique to this control — nothing else in the app uses it, so it's unambiguously "the ownership filter" from any glance.

## Files changed

- `web/assets/wd-tools.css` — new `.btn-icon-only` variant; Cloud Manager `.btn-secondary` outlined treatment; Cloud Manager `#liveBtn` toggle styling with `::before` dot indicator; `.header-actions .btn` rule split so color variants (`.btn-blue`, `.btn-green`, `.btn-red`) keep their identity while `.btn-secondary` gets the translucent header style. Light-mode override for `.header-actions .btn-secondary` added.
- `web/cloud.html` — Settings button reclassed `.btn-blue` → `.btn-icon-only`; Refresh reclassed `.btn-blue` → `.btn-secondary`; Live dropped `.btn-secondary` to use the dedicated `#liveBtn` toggle rules. Version chip `v3.15 → v3.16`.
- `web/pages/hosted-index.html` — suite chip `v1.8.5 → v1.8.6`.

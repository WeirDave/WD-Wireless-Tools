Two visual tweaks: tighter toolbar spacing + extras' amber wash removed.

Suite `v1.8.24 → v1.8.25`. WD Cloud Manager `v3.34 → v3.35`.

## 1. Toolbar row is tighter and more balanced

Dave: *"there is more padding above the search bar and delete button than there is below."*

Measurement showed the toolbar itself was symmetric (6 px above, 7 px below), but the combined empty space between the dashboard cards and the search row was way bigger than the space below the search row — dashboard's `padding-bottom: 8px` + toolbar's `padding-top: 4px` = 12 px of dead air above vs 4 px below.

Fix:
- Dashboard `padding: 8px 12px → 6px 12px 4px` (top 6, sides 12, bottom 4). Shrinks the wasted space between the count-cards and the toolbar.
- Toolbar `padding: 4px 12px → 3px 12px`. Slightly shorter row.

Result:
- Space between dashboard and search row: **5 px** (was 12 px — over half gone).
- Toolbar height: **35 px** (was 37 px).
- Search bar: 5 px above / 6 px below — symmetric within 1 px.
- Delete-all button: 4.2 px above / 5.2 px below — symmetric within 1 px.

## 2. Extras' amber wash removed — outline only

Dave: *"It's just too much orange, take the background out of the extra and just have that card be outlined in amber like you have it now."*

`.dup-item.is-extra` no longer applies the amber-wash `background`. Extras keep:
- 3 px amber inset outline (`box-shadow`).
- Bold name text.
- Amber side icon.

Row body sits on the same `--surface` as matched rows. Amber is now purely an outline signal — no room-filling color competing with the panels above.

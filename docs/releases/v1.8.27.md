Suite-wide visual language pass. Every tool now shares the same header shape, the same card hover, the same button silhouette, and the same theme-toggle pill in the exact same spot on every page. Plus the Duplicates cleanup that landed alongside it.

Suite `v1.8.26 → v1.8.27`. WD Cloud Manager `v3.36 → v3.38`.

## 1. Duplicates cleanup

- **Cloud project dates were missing** on Duplicates rows because `_parse_cloud_mtime` was only checking flat keys. It now also reads `pr["history"]["modifiedAt"]` and `pr["history"]["createdAt"]`, which is where Ekahau's API actually puts them.
- **LARGEST / NEWEST pills removed.** They added noise without adding signal — the columns already show size and date.
- **Size and date columns are now right-aligned and fixed-width** (90 px / 105 px), so numbers line up cleanly across every row.
- **Row cards use blue/green bookends** (8 px, rounded on the outer corners) with a 1 px grey `--border` on top and bottom — the row now reads as a fully-framed card, not a floating stripe.
- **Extras get a 2 px amber inset outline** instead of a full amber background wash — cleaner distinction, less visual noise.
- **Cluster header:** `#0668D9` blue at 10 % on `--surface`. **Info panel:** amber at 10 % on `--surface`. Both use `linear-gradient(rgba, rgba), var(--surface)` so the translucent tint composites over a consistent base.
- **`.dup-actions-bar .btn-sm`** cleaned up (padding + pill radius).

## 2. Unified headers across all five tools

Every tool now uses the same `.topbar` sizing: `min-height: 56 px`, `padding: 0 16 px`, `h1` 22 px, `.ver` 14 px. Report and Walls share the same rule via `.dz-topbar` and `.report-toolbar`.

Light-mode topbar: **20 % `#0668D9` blue wash on `--surface`** — reads clearly as blue rather than grey. Text stays `var(--text)` for legibility.

Tab titles: `var(--text-2)` on `var(--surface)` in light mode, with `var(--border)` on the sides. No line between the topbar and the tab strip on Cloud.

Hamburger: added `[data-theme="light"]` overrides — dark bars + `--border` — so it's visible in both themes.

## 3. Theme toggle in the exact same spot on every page

Every page (Cloud, Report, Scale, Squirrel, Walls) uses the same markup:

```html
<button class="theme-toggle" onclick="toggleTheme()">
  <span class="ico">🌙</span><span class="txt">Dark</span>
</button>
```

Positioned as the last element in `.topbar-right` / `.header-actions` — far right, even padding. `WD.syncThemeUI` now iterates every `.theme-toggle` on the page via `querySelectorAll`, so Report and Walls (which have two instances — landing + workspace toolbar) both update in sync.

## 4. Universal card hover — every card, every page, one behavior

Home `.card`, Squirrel `.pick-card`, Scale `.scale-panel`, Cloud `.dash-card`, Walls `.wall-card`, and Walls `.hotkey-slot` all share the same hover:

- `transform: translateY(-4px)`
- `box-shadow: 0 16px 40px rgba(0, 0, 0, 0.28)`
- `border-color: <variant>` — blue / green / amber / purple, or per-card `--wall-color` / `--slot-color` set inline from JS
- `background: var(--surface-raised)` — resolves to `#f6f8fa` in light, `#252830` in dark. Identical hover background everywhere.

Persistent top-flare removed from `.pick-card` and `.scale-panel` — they now match home's clean-until-hover pattern. On hover, the accent lights up at the border, not as a always-visible tab.

## 5. Workspace background is pure white in light mode

`[data-theme="light"] body.tool-organizer, body.tool-scale, body.tool-walls { --bg: #ffffff; }`. Before, these tools sat on a slightly grey `--bg` while home was pure white — the cards read differently side by side. Now every workspace uses the same pure-white background as the home page.

## 6. Yellow number badges → `--amber-dark` in light mode

`#F0C050` fails WCAG on white. `.wall-keybind-badge`, `.hotkey-num`, `.keybind-option .num`, and `.dropzone-steps li::before` all get `[data-theme="light"]` overrides that swap yellow for `var(--amber-dark)` (`#b45309`). Dark-mode yellow is untouched — it reads fine on the dark background.

## 7. Every button is a pill, and every button lifts on hover

**One silhouette:**
- Base `.btn`: `border-radius: 8 px → 999 px`
- `.card .open-btn` (home landing): `8 px → 999 px`
- `.header-actions .btn` (Cloud / Report): `5 px → 999 px`
- `body.tool-cloud .btn` (Cloud toolbar): `3 px → 999 px`
- `body.tool-walls .btn` (Walls save / apply / defaults / etc.): `5 px → 999 px`
- `.dup-actions-bar .btn-sm`: `5 px → 999 px`
- `.login-btn`: `6 px → 999 px`
- `.btn-icon` and `.btn-icon-only` (× close, ⚙ gear): `4 px → 50 %` (perfect circle)

**One hover:**
- `.btn:hover:not(:disabled)`: `translateY(-2 px)` + `box-shadow: 0 6px 18px rgba(0, 0, 0, 0.22)`
- `.btn:active:not(:disabled)`: press-back to `y = 0` with a smaller settled shadow

Variant color-darkening (`.btn-blue:hover → var(--blue-dark)`, etc.) still stacks on top — the 3 D lift is orthogonal to the color, so a solid-blue primary and a translucent secondary both pop the same way on hover.

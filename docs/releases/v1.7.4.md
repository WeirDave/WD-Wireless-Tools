# WD Report v1.3 — Visible "Open another .esx" buttons

v1.7.3's file-swap options were all hidden: hamburger menu → "Open Another File", or click the filename pill in the top-left. Neither was discoverable. v1.7.4 adds two visible buttons that both call the same `loadNewFile()` picker.

## What changed

**Top-right toolbar** — a new grey `📁 Open another…` button sits next to the blue `🖨 Print / Save PDF` button. Same treatment as Quick Walls' toolbar-level actions.

**Sidebar bottom** — a full-width `📁 Open another .esx` button pinned above the "runs locally" footer. Useful after you've been reading the report and are looking at the sidebar for the next action.

Both call the existing `loadNewFile()`, which resets the hidden `<input type="file">` and re-triggers the picker. The dropzone/workspace swap logic already handles a fresh drop cleanly — no state carries over from the previous file.

The filename pill (top-left) and hamburger-menu entry stay in place — three ways to reach the same picker.

## Files changed

- `web/report.html` — added the two buttons, page version chip → `v1.3`.
- `web/assets/wd-tools.css` — new `.rep-sidebar-actions` container (pushed to the bottom via `margin-top: auto`) and `.rep-open-btn` full-width button styling. Reduced `.rep-foot` top padding since it's now below an action divider instead of being the divider itself.
- `web/pages/hosted-index.html` — version chip → `v1.7.4`.
- `README.md` — Report sub-version → `v1.3`.

## Verification

Hard-refresh `/report`. Drop an `.esx`. Two new buttons appear:

1. Top-right toolbar, left of the Print button — grey with a folder icon.
2. Sidebar bottom, above the "runs locally" note — full-width grey with a folder icon.

Click either → native file picker opens → pick another `.esx` → the report re-renders. Cover, logo, and sidebar options all carry over (they aren't tied to the file).
